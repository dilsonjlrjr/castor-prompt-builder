How it works

Six steps to a structured prompt

No configuration. No API keys. Just a wizard that extracts what matters and fills the gaps.

Step 1
📐

Select a model

Choose a prompt structure (RTF, RACE, RISEN or CREATE) based on the complexity of your task.

Step 2
🎭

Pick role(s)

Search 47 professional roles. Select multiple — their skills and tone merge automatically.

Step 3
✍️

Write your narrative

Describe your task in plain language. No special syntax. The engine maps it to the template fields.

Step 4
🔍

Fill the gaps

Fields the engine couldn't auto-fill become interactive questions. Required ones are flagged.

Step 5
🗂️

Define phases (optional)

Structure the execution into numbered phases. Each gets a title and description injected into the prompt.

Step 6
✅

Prompt saved

Your structured .md prompt is saved to prompts/, ready to paste into any LLM.

Prompt models

Four structures. One right fit.

Each model is a proven framework for structuring LLM instructions. Pick the one that matches your task's complexity.

RTF

Role · Task · Feature

Simple & direct

Best for focused, single-output tasks where you know exactly what you want.

RACE

Role · Action · Context · Expectation

Rich context

Best for complex briefs where background and deliverable clarity matter most.

RISEN

Role · Input · Steps · Expectation · Narrowing

Step-by-step reasoning

Best for long tasks with explicit constraints and multi-stage reasoning requirements.

CREATE

Context · Role · Examples · Audience · Tone · Expectation

Creative output

Best for content creation with audience definition and tone control.

Features
